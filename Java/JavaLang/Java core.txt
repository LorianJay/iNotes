1.java语言的特性:
java是面向对象语言.注重于数据之间的交互关系.摒弃了C++的多继承和容易造成程序错误的内存管理.java是编译语言.
java为什么可以实现跨平台?
在计算机中有这么几个关键的东西:硬件、操作系统、编程语言.
硬件被操作系统管理,C和C++是直接和操作系统进行交互,所以依赖操作系统.而java的class文件依赖于JVM虚拟机.java编写的代码是和JVM虚拟机进行交互的,然后jvm虚拟机在和操作系统进行交互.这样对于java程序员而言只需要和代码打交道,在不同的操作系统上只需要安装一个JVM虚拟机即可,代码原封不动.所以基于JVM虚拟机的kotlin语言也是跨平台的.


2.深入理解类加载全过程.

---------------------
比如这里有Hello.java文件,调用javac命令对java文件进行编译编译成class字节码文件.这个字节码文件只有符合虚拟机的规范才可以被执行.
当我们执行java Hello 命令时发生的哪些步骤?

首先将字节码文件通过类加载器加载到虚拟机的方法区当中.					//步骤一:加载
然后对这个字节码文件进行验证,看是否满足虚拟机加载的规范.				//步骤二:验证
把类的类变量初始化为初始化值.如果有final关键字修饰,那么直接初始化为这个变量该有的值.	//步骤三:准备
											//步骤四:解析
按照顺序将静态变量初始化应有的值,执行静态代码块.					//步骤五:初始化
找到main方法,开始执行.只有当一个类被实例化的时候,会先初始化实例变量为JVM默认值,之后再初始化为该有的值.而且优先于普通代码块(构造器)所以第七个步骤是建立在被new的情况下								//步骤六:使用
											//步骤七:卸载

什么是类构造器方法和对象构造器方法:
类构造器方法就是在步骤四这个阶段执行,它会保证先执行当前类的父类的类构造方法.这个方法JVM帮助我们执行,执行的具体内容就是将当前类的静态变量和静态代码块按顺序融为一体执行.
对象构造器方法就是当一个类被实例化的时候,执行当前对象的构造方法.

什么是实例变量、实例方法什么是类变量:
实例变量就是有没有static修饰的变量
类变量就是有static修饰的变量.

将类变量或实例变量初始化:
byte 0
short 0
int 0
long 0L
float 0.0f
double 0.0d
char '/u000'	实际上就是''	中间没有值
boolean false
如果是引用类型:null

为什么static方法无法访问非静态属性(实例变量):
因为实例变量只有在被new(或者是被调用构造器)的时候才会创建.静态方法被加载的时候就已经被加载到方法区了,比如在静态代码块中访问一个实例变量,这个变量还没有被创建你就能访问了?此时你访问的实例变量还根本不存在.

注意:如果一个类被调用了构造器,那么不管是怎么调用的,其性质就相当于被new了.

----------------------
public class Hello{
	public static void main(String[] args){
		Hello hello = new Hello();
	}
}

运行这段代码的过程:
JVM会自动寻找到主函数的main方法.为main方法在栈内存中开辟一个栈帧.这个栈帧做了哪些事?
1.在线程池中声明了一个对象 Hello hello;
2.在JVM的堆内存申请了一片空间地址,然后将Hello相关信息如:实例变量、实例方法从方法区加载到堆内存中.
	在堆内存中创建对象的时候执行顺序:
	2.1加载实例信息进入开辟的内存中
	2.2执行对象的构造方法
	堆内存中开辟的对象结构:对象由对象的头部信息和实例的信息组成.
	头部信息:
	1.对齐填充
	2.持有指向方法区的指针(一些静态方法和静态属性是存储在方法区当中的,对象的静态内容实际上是指向方法区的静态内容)
	3.描述信息(持有当前对象的锁的线程的id和持有对锁线程的个数,在gc中存活的声明周期,偏向锁的标志.)
		偏向锁:当线程对此对象上锁后,执行完毕如果下一次访问该对象的线程也是上一次的线程那么不需要此线程重新上锁.
3.另外线程栈(栈帧)的大小默认为1M.

public class Father {
    static {
        System.out.println("Father静态代码块");
    }

    {
        System.out.println("Father普通代码块");
    }

    public Father() {
        System.out.println("Father构造方法");
    }
}

class Son extends Father{

    static {
        System.out.println("Son静态代码块");
    }

    {
        System.out.println("Son普通代码块");
    }

    public Son() {
        System.out.println("Son构造方法");
    }

    public static void main(String[] args) {
        new Son();
    }
}

输出结果:
Father静态代码块
Son静态代码块
Father普通代码块
Father构造方法
Son普通代码块
Son构造方法

原因:

总结来说当每个类进行到步骤五之前,都会去检测该类是否有父类.如果有父类优先加载父类的静态代码块,然后再加载子类的静态代码块.在步骤六使用的时候 会调用构造器,构造器默认添加了super关键字,所以会调用父类的构造器,记住一个我总结出来的特性,super是构造器执行的第一行代码,当这行代码执行完毕之后直接去调用当前类的普通方法块,然后再接着执行super后面的代码.

4.常量池和JVM原理(重点是常量池)

常量池包括:符号引用(以表的形式存在)

常量表类型			标志值(表的大小)(单位byte)	描述
CONSTANT_Utf8			1				UTF-8编码的Unicode字符串(保存字符串的Unicode编码)
CONSTANT_Integer		3				int类型的字面值	
CONSTANT_Float			4				float类型的字面值
CONSTANT_Long			5				long类型的字面值	
CONSTANT_Double			6				double类型的字面值
CONSTANT_Class			7				对一个类或接口的符号引用
CONSTANT_String			8				String类型字面值的引用
CONSTANT_Fieldref		9				对一个字段的符号引用
CONSTANT_Methodref		10				对一个类中方法的符号引用
CONSTANT_InterfaceMethodref	11				对一个接口中方法的符号引用
CONSTANT_NameAndType		12				对一个字段或方法的部分符号引用

当虚拟机加载一个类时,当这个类里面有字符串时.JVM会将该类的所有字符串转换为其对应的Unicode编码.然后把这个Unicode编码放到CONSTANT_Utf8这张表里.然后会为该字符串生成一个TAG(标记)这个标记对应一个索引,存放到CONSTANT_String表里.然后再根据CONSTANT_String这个表里的索引,指向CONSTANT_Utf8表里具体的Unicode编码.
所有的字面值都是存储在这张表里面的,什么是字面值?上面说的字符串是其中的一种.实际上类名、方法名、形参名称所有非java语法,开发人员可自行定义的文字都是字面值.

·用这个知识来解决问题:
问题1:
String str1 = "String";
String str2 = "String";
System.out.println(str1==str2);		//输出true

原因:
当JVM加载到str1时.会将str1对应的Unicode编码存放到CONSTANT_Utf8表里,然后在CONSTANT_String表里生成一个Tag 这个Tag又指向CONSTANT_Utf8表对应的Unicode编码.当JVM加载到str2时也会生成一个对应的TAG,然后这个TAG指向和str1一样的Unicode编码.所以这两个的内存地址相同.

问题2:
String str1 = "String";
String str2 = new String("String");
System.out.println(str1==str2);			//输出false

原因:
要区别这两个语句的本质区别.刚才的(上面的)str1和str2是直接指向CONSTANT_String表的.现在str1还是直接指向CONSTANT_String表的.但是现在的str2所指向的是对内存当中的String对象.然后这个String对象再指向方法区.所以现在str2的指针是指向堆内存的.


1.java中==是用来比较数据的地址的(除了基本类型)
2.equals方法默认与==一样,也就是说从Object对象里继承过来的方法默认比地址
3.在java中所有对象的超类都是Object.

什么是超类?
假如类A继承了类B,类C继承了类B.那么类A就是类C的超类.

·toString:
System.out.println(new Demo());		//当我们输出当前对象 输出的值是当前对象的equals方法的返回值.

·native:
在java中方法修饰符native表示这个方法是通过JNI来调用C语言或C++来执行的.
这些方法存储在本地方法栈.之前所说的栈以及开辟栈帧实际上就是线程栈.

·intern:
String str1 = "String";
String str2 = new String("String");
System.out.println(str1==str2.intern());		//输出true

原因:
虽然之前将str1和str2指向的内容不同.但是String的intern方法返回值是常量池里面字面值的地址(也就是将CONSTANT_String表里对应的索引值(字面值)返回).如果常量池里面没有这个索引值,那么先把这个字面值放到常量表里面之后再返回常量表的地址.(其实也很好里面,要调用该对象的方法之前肯定想声明内容)

·拼接字符串:
String str1="String";
String str2 = "Str"+"ing";
System.out.println(str1==str2);			//输出true

原因:
像这种静态的拼接,就认为JVM是直接将拼接后的内容存放到CONSTANT_String表的.然后str1和str2指向同一个CONSTANT_Utf8表里的Unicode编码.

String str1="String";
String str2="Str";
String str3=str2+str1;
System.out.println(str3==str1);		//输出false

原因:
动态的拼接在运行时通过stringbuilder拼接后返回的toString().所以此时str3指向的是stringbuilder的内存地址.

·关于类变量的坑:
class TestStatic {
	static String name = "你好";
}

此时name类变量可以被任何类访问到,可以直接通过 类名.name就直接对该变量进行修改.如果该类被创建实例 TestStatic testStatic = new TestStatic();
虽然是实例了一个新的对象,但是还记得之前所说的吗?静态变量是存放在方法区当中的,所以对于name类变量而言,栈空间中的testStatic指向堆空间的实例地址,堆空间中的name类变量又是指向方法区当中的.所以对于这个变量只要一处动处处全动,不代表一个新的对象 这个对象中的name属性就有新的内存地址.

7.权限

public:所有类对象都可以访问
protected:只有该类和子类可以访问.该类和子类的实例都可以访问.
private:只有该类可以访问.其子类也不可以访问.该类的实例(对象)不可以访问.

通过字节码我们可以看到我们在平时调用private的时候是通过this关键字来调用的.而private方法的信息保存在CONSTANT_Methodref这张常量表里面,也可以总结出JVM中的
invokespecial指令是用来调用类的构造器和私有方法.
invokevirtual指令调用实例方法.

一个方法在JAVAC编译的时候就已经确定了这个方法的描述也就是方法的访问权限和方法栈深度、本地变量个数和参数个数都已经确定好了,所以我们在调用方法压入栈帧的时候,栈帧的大小都在javac编译的时候确定好了.

9.重写+多态

class A{
	public void out(){
		System.out.println("A");
	}
}

class Demo extends A{
	public void out(){			
		System.out.println("Demo");
	}
	public static void main(String[] args){
		A a = new Demo();
		a.out();
	}
}

输出结果:
Demo		//这是多态的基本知识

·子类重写父类方法,那么子类重写的方法的权限一定要大于父类的权限.原因:如果这里Demo类的out方法的权限是private,那么在使用多态的时候out方法是调不到的.所以子类的权限一定要大于父类.

·并且子类重写的方法也不可以加上静态的修饰符,但是可以加上final常量修饰符.而且如果父类方法有异常,那么子类继承重写的方法也会默认有父类的异常.
而且子类只能抛出父类抛出异常的子类异常,子类抛出的异常不能比父类高.	

·底层分析:类A和类Demo的out方法都是存放在方法区的CONSTANT_Methodref这张表里面.这两个out方法的指针分别指向不同的TAG.当Demo对象被实例化的时候,Demo的out方法是指向自已的TAG的.之前说过方法的名称是存储在CONSTANT_Utf8这张表里面的.

·为什么ArrayList不但继承了AbstractList又实现了List接口?
主要的原因是为了实现JDK的动态代理.想想如果ArrayList没有实现List接口.那么根据继承关系你使用的就是多态.List list = new ArrayList();这样是可以的.但是在JDK动态代理里面,如果ArrayList没有实现List.那么就无法获取到List.现在ArrayList实现了List相当于是接口的实现类.

·ArrayList:
集合默认大小是10.如果大小不够用,按照底层算法可知,ArrayList是按照原大小的0.5倍增长的.建议给容器一个初始的大小.不要太小也不要太大.太小的话,数组的扩容是很消耗内存的,太大的话浪费空间.由于ArrayList底层是通过数组实现的,所以ArrayList又被称之为线性表,便于查找(直接根据索引进行查找),访问时间复杂度o(1).但是不便于修改.应用场景:少改多查.

    }
}

然后在EnumException的构造器中去解析这个异常,并将枚举元素的message传给父类构造器.

13.6泛型的擦除
泛型的擦除分为无限制擦除和有限制擦除,泛型的擦除使得每个类在编译的时候具体的类型是确定的,并且其类型只有两种情况(实际上大部分是Object类型). 运行时所有的类型都已经确定了!
所以编译期间做的操作非常多.

无限制擦除:
public class<T> Box{				
    private T value;
    public void setValue(T value){
        this.value = value;
    }
    public T getValue(){
        return this.value;
    }
}
编译之后为:
public class Box{				
    private Object value;
    public void setValue(Object value){
        this.value = value;
    }
    public Object getValue(){
        return this.value;
    }
}

有限制擦除:
public class<T extends Number> Box{				
    private T value;
    public void setValue(T value){
        this.value = value;
    }
    public T getValue(){
        return this.value;
    }
}
编译之后为:
public class Box{				
    private Number value;
    public void setValue(Number value){
        this.value = value;
    }
    public Number getValue(){
        return this.value;
    }
}

同样对于泛型方法的类型擦除,也分为有限制和无限制类型擦除:
public <T> T getValue(T value){
}
编译之后:
public Object getValue(Object value){
}

public <T extends Number> T getValue(T value){
}
编译之后:
public Number getValue(Number value){
}

最重要的桥接方法:
public interface Info<T>{
    T info(T var);
}
编译后:
public interface Info{
    Object info(Object var);
}

假设现在有个类实现了该接口,并且指定了泛型类型,如下:
public class InfoImpl implements Info<Integer>{
    @Override
    public Integer info(Integer var){
        return var;
    }
}
编译后:
public class InfoImpl implements Info{
    public Integer info(Integer var){
        return var;
    }
    // 桥接方法,保持接口和类的实现关系
    @Override
    public Object info(Object var){
        return info((Integer)var);
    }
}
