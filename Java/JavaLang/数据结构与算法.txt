·21.有些题目是给你一些个数,然后让你将这些数组成一个最大数.这种题目无非就是可以通过某种形式得到一个数组.然后根据这个数组里面的元素去组成一个最大数,思路:
首先我们需要将这个数组排序,例如将数组[3,30,34,5,9]排序,我们最终应当得到的数组是[9,5,34,30,3](排序规则是按照最高位的大小进行排序的).
所以我们要定义我们自已的比较器.
Arrays.sort(nums, (o1, o2) -> {
    //2147483647 10
    byte[] array1 = new byte[10];
    byte[] array2 = new byte[10];
    int index1 = 0;
    int index2 = 0;
    while (o1 > 0) {
        array1[index1++] = (byte) (o1 % 10);
        o1 /= 10;
    }
    while (o2 > 0) {
        array2[index2++] = (byte) (o2 % 10);
        o2 /= 10;
    }
    for (int i = 0; i < 10; i++) {
        byte pop1 = array1[i];
        byte pop2 = array2[i];
        if (pop1 != pop2) {
            return pop2 - pop1;
        }
    }
    return 0;
});

3.斐波那契数列(实际上就是动态规划)

什么是斐波那契数列?
f(1)=1 f(2)=1 f(n)=f(n-1)+f(n-2)

什么时候使用斐波那契算法?
也就是当这一次的值依赖于上一次的结果或者上几次的结果.

怎么求斐波那契数列?
不是真的一个一个地求值,还有更蠢的用递归求值,一直递归到f(1) f(2)层面然后根据执行的次数得出最后的值.
这种题目应该怎么解?
将上两次的结果保存起来,然后根据上两次的结果相加,对于下次求值而言,这次的f(n)变为下次的f(n-1),这次的f(n-1)变为下次的f(n-2),伪代码:

System.out.println(Integer.toBinaryString(-5));		//其结果是11111111111111111111111111111011
二进制转十进制的时候还是用的原码(这个是废话),补码转原码时,正数的补码就是原码.负数的补码符号位不动,其余所有位取反,最后+1(也是+1不是减!)
byte b = -127;		//补码为10000001(原码为11111111)
在byte转int的时候为了确保转型之后的十进制不变,Java是这么干的:
对byte进行高位补码补1,结果为1111 1111 1111 1111 1111 1111 1000 0001(这个就是转为int类型之后的补码了).你会发现将其转为原码后结果就是1000 0000 0000 0000 0000 0000 0111 1111(也就是int类型表示-127)
但有的时候我们希望转型之后,希望让int的补码和byte的补码一致,怎么办?
让byte&0xFF(0xFF代表11111111也就是取byte的最低位8位)
首先还是对byte进行高位补码补1,结果为1111 1111 1111 1111 1111 1111 1000 0001(这个也是补码)然后将该值的后8位保留,前面的所有位置0,所以现在该补码也就是原码即0000 0000 0000 0000 0000 0000 1000 0001(也就是int类型的原码表示十进制数129)
这里由于int这么搞之后,int的补码就等于原码了(因为是正数),相当于用整个int的后8位表示byte的8位补码.
所以在Java当中所有的位操作都是对补码的操作.

那么这个有什么用呢?在京东支付中,可以用于字节数组转2位16进制的String,代码如下:
public static String bytes2Hex(byte[] bts) {
	String des = "";
	String tmp = null;
	for (int i = 0; i < bts.length; i++) {
		/*
		 保障byte->int补码不变,并转为16进制字符串.1字节的byte(8位二进制)可以用2个16进制表示
		 toHexString:将二进制转为16进制字符串(但是这个方法参数只支持int,直接把byte传进去是不行的,因为byte转int的规则是保证十进制值一致)
		 所以这里为了保障转型后补码一致,采用了&0xFF这种方式,另外如果16进制低位不够就补上一个0(保障这里的16进制是两位)
		 */
		tmp = Integer.toHexString(bts[i] & 0xFF);
		if (tmp.length() == 1) {
			des = des + "0";
		}
		des = des + tmp;
	}
	return des;
}

同样2位16进制的String转byte字节数组的代码如下:
public static byte[] hex2byte(String s) {
	byte[] b = s.getBytes("UTF-8");
	// 因为补位,所以该字符串一定是偶数
	if (b.length % 2 != 0) {
		throw new IllegalArgumentException("长度不是偶数");
	}
	byte[] b2 = new byte[b.length / 2];
	for (int n = 0; n < b.length; n += 2) {
		String item = new String(b, n, 2);
		/*
		正因为在bytes2Hex方法中,byte转int的时候保证了byte的补码和int的补码一致(并且int的补码就是原码)
		这里将16进制转回为int的时候,转出来的int的补码和原码一致,再转为byte的时候,高位截断使得byte的补码和int的补码一致,得到byte原来的补码.
		 */
		b2[n / 2] = (byte) Integer.parseInt(item, 16);
	}
	return b2;
}

8.队列+广度优先算法(BFS)
·一般求什么最短路径,最少长度,最少个数之类的,凡是和少相关的要能联想到广度优先算法.(另外二叉树的层次遍历就是使用广度优先算法,包括图的两种遍历方式)
·但是广度优先算法存在效率问题.
·什么是广度优先算法?
BFS:广度优先算法(宽度优先搜索)
BFS算法的一个常见应用是找出从根节点到目标节点的最短路径(长度).BFS算法实际上是和队列结合使用的.

·计算出A到G的最短路径长度:

    B─E
 ╱  ╱
A─ C─F
 ╲    |
    D─G

回合1:将A节点添加至队列->判断A是否是目标节点G,如果是则返回回合数-1,否则继续->将A的所有右节点添加至队列(B C D)->删除A节点
回合2:判断B是否是目标节点G,如果是则返回回合数-1,否则继续->将B的所有右节点添加至队列(E)->删除B节点->判断C节点的是否是目标节点,如果是则返回回合数-1,否则继续->将所有C的右节点添加至队列(F)->删除C节点...
回合3:判断E节点是否是...在判断到G的时候成功找到,返回回合数-1,所以结果为2.

注意新添加的节点只会在下一个回合中处理.
大体的流程图:
A
A->B->C->D

B->C->D
B->C->D->E
C->D->E->F
D->E->F->G

E->F->G
F->G
G		//找到节点G

BFS模板二叉树模板:
    public int findBottomLeftValue(TreeNode root) {
        Queue<TreeNode> queue = new ArrayDeque<>();
        queue.add(root);
        int result = root.val;
        int next = 0;
        int count = 1;
        int max = 0;
        while (!queue.isEmpty()) {
            TreeNode poll = queue.poll();
            if (max == count) {
                result = poll.val;
            }
            count--;
            if (poll.left != null) {
                queue.add(poll.left);
                next++;
            }
            if (poll.right != null) {
                queue.add(poll.right);
                next++;
            }
            if (count == 0) {
                count = next;
                max = next;
                next = 0;
            }
        }
        return result;
    }

但是数组的空间效率不是很好,因为数组创建的时候需要指定数组的容量大小,然后根据大小分配内存,这很容易导致会有空闲的区域没有被用到.

·有时候我们无法确定数组的具体大小,可能只有一个范围的概念,这时候我们可以使用List来存储数据,因为List是可变大小的集合,到时候存了多少个元素数组的大小就是多少个.然后再通过list.toArray(new T[]);将list转为数组.但是这种方式无法将list转为基本类型的数组.那么有派生出另外一种解决方法,list集合的内容已经确定,创建一个新数组,数组的大小就是集合的size.然后遍历集合,填充数组.但这种做法效率不高,你永远不应该遍历集合只是因为将其转换为数组!!!

·如何使用循环进行遍历?一种可能就是使用栈结构,栈从某种意义上来讲就是递归,递归和栈有的共同特点就是深!

·二叉树节点的删除问题(暂定):
当你删除树中的节点时,删除过程将按照后序遍历的顺序进行.也就是说,当你删除一个节点时,你将首先删除它的左节点和它的右边的节点,然后再删除节点本身.

·二叉树运用递归解决数的问题的两种方式:
1.自顶向下遍历:类似于前序遍历,从根节点往下遍历,每次传入根节点的子树
2.自底向上:类似于后序遍历,从叶节点向上return.先找到二叉树的叶节点,然后根据叶节点的返回值,一层层往上处理.

当遇到树问题时,思考以下两个问题:
自顶向下:
(1)你能确定一些参数,从该节点自身解决出发寻找答案吗?
(2)你可以使用这些参数和节点本身的值来决定什么应该是传递给它子节点的参数吗?
如果可以确定,那么就尝试使用自顶向下来解决问题.
自底向上:
对于树中的任意一个节点,如果你知道它子节点的答案,你能计算出该节点的答案吗?如果答案是肯定的,那么尝试使用自底向上遍历.

通俗来讲就是:在二叉搜索树中,该节点的左子树中的最大值就是该节点的前驱节点.


·什么是后续节点?
后继节点的val值应该大于当前节点的val值并且后续值最小的节点

通俗来讲就是:在二叉搜索树中,该节点的右子树的最小值就是该节点的后继节点

·如果你想到了用前驱节点和后继节点来解决二叉树的问题,那么一定要考虑前驱节点还有做子节点的情况,后继节点还有右子节点的情况.


2.二叉搜索树

·什么是二叉搜索树?
每个节点中的值必须大于(或等于)其左侧子树中的任何值,但小于(或等于)其右侧子树中的任何值.

·二叉搜索树的节点查询和插入都不是特别难,这里主要注重删除操作.
二叉搜索树的删除主要分为三种情况:

i.当这个节点为叶节点时(也就是没有子节点)
这种情况最简单只要让该叶节点的父节点不指向该叶节点即可.

ii.当这个节点只有一个子节点时
实际上只需要将该节点的子节点替换该节点即可.(也就是让该节点的父节点指向该节点的子节点)

iii.当这个节点有两个字节点时
我们需要将当前节点的值替换为该节点的前驱节点的值或者是该节点的后继节点的值,然后再将前驱节点或者后继节点删除.注意需要考虑前驱节点和后继节点还有子节点的值的情况.

7.二叉树中的度

·什么是度?
首先说说什么是度:通俗的讲二叉树中连接节点和节点的线就是度,有n个节点,就有n-1个度,节点数总是比度要多一个,那么度为0的节点一定是叶子节点,因为该节点的下面不再有线;度为1的节点即:该节点只有一个分支,同理度为2的节点就是有两个分支.在二叉树中不可能存在度为3或大于3的节点!

度取决于当前节点的子节点个数.度只是一种标识

·计算公式:

k:总度数
k+1:总节点数
n0:度为0的节点个数
n1:度为1的节点个数
n2:度为二的节点个数
根据二叉树中度和节点的守衡原理，可列出以下一组方程：
k=n2*2+n1;
k+1=n2+n1+n0;
将上面两式相减得到:n0=n2+1;

结论包括但不限于:度为0的节点数为度为2的节点数加1，即n0=n2+1

8.线索树
对于传统的二叉树,假设二叉树有n个节点,则该二叉树一共有2n个指针,因为每个节点都有两个指针所以一共有2n个指针.
并且除了根节点之外,每个节点都被一个指针所指向,所以一颗二叉树一共有n-1个节点被指针所指向.那么用总指针数减去被使用的指针数,得到一共有2n-(n-1)=n+1个空指针,例如:
    1
  ╱ ╲
2    3
节点2和3分别被根节点1的左指针和右指针指向,但是节点2和3的左右指针没有指向任何节点是空节点.
要知道在Java中一个指针变量需要开辟4字节的内存空间,这些不指向任何节点的空指针会造成内存空间的浪费,为此我们提出了线索树的概念.
我们可以利用这些空指针存放相应的前驱节点和后继节点的地址,这些指向前驱节点和后继节点的指针被称作线索.
有如下规定:
当某节点的左指针为空时,让该指针指向对应遍历序列的前驱节点,当某节点的右指针为空时,让该指针指向遍历序列的后继节点.如何区分左指针指向的是左孩子还是前驱节点以及右指针指向的是有孩子还是后继节点呢?我们通过增加两个标识位来区分这两种情况.
有布尔值isLeftChild和isRightChild 当isLeftChild为true代表当前指针指向的是左子树否则是前驱节点,右节点的逻辑同理.
按上述方法在每个节点上添加线索的二叉树称作线索二叉树.

在构造线索树的时候,必须先求出线索树的遍历序列,不同的遍历方式得到的线索树也是不同的.
为了使算法设计方便,在线索二叉树中再添加一个头结点,头结点的data成员为空,left节点指向二叉树的根节点,isLeftChild设置为true,right指向遍历序列的尾结点,isRightChild设置为false.
在比较的过程总检查当前节点的左右指针域是否为空,如果为空,将他们改为指向前驱节点或后继节点.

9.线段树
假设现在有一个数组array,我们有一个query方法是将某段区间的元素加和,假设该区间为[L,R],假设L-R中间有N个元素则该算法的时间复杂度为O(n),另外我们还有一个update方法是修改数组中某个元素的值,那么它的时间复杂度为O(1).
假设现在要对query方法和update方法做n多次操作,那么由于query方法是O(n)最后会导致算法的时间复杂为O(n^2).那么有没有一种方法使得query方法的时间复杂度为O(1)呢?
现在创建一个sum_array数组(长度和array一样大),有sum_array[i] = Σ(n=0...i)array[i]
那么任意区间[L,R]的元素的加和,只要将sum_array[R]-sum_array[L-1]的结果返回即可,此时query方法的时间复杂度为O(1),但此时update方法的时间复杂度却变为O(n).所以当我们query和update操作非常多的时候无论使用哪种方法,最后总体的速度都不会特别快.
那么使用线段树可以使得query和update的时间复杂度都为O(logn)

13.二分法查找

·使用二分法的一大特点就是有序!!!

·模板1:左闭右闭 [left,right]
int binarySearch(int[] nums, int target){
  if(nums == null || nums.length == 0)
    return -1;

  int left = 0, right = nums.length - 1;
  while(left <= right){
    int mid = left + (right - left) / 2;
    if(nums[mid] == target){ return mid; }
    else if(nums[mid] < target) { left = mid + 1; }
    else { right = mid - 1; }
  }
  return -1;
}



·模板2:左闭右开 [left,right)
int binarySearch(int[] nums, int target){
  if(nums == null || nums.length == 0)
    return -1;

  int left = 0, right = nums.length;
  while(left < right){
    int mid = left + (right - left) / 2;
    if(nums[mid] == target){ return mid; }
    else if(nums[mid] < target) { left = mid + 1; }
    else { right = mid; }
  }
  if(left != nums.length && nums[left] == target) return left;
  return -1;
}

    while (left + 1 < right){

            left = mid;

14.时间复杂度的计算
·时间复杂度通过Big O来标识
·时间复杂度实际上就是随着数据量的扩大,计算时间变化的规律.
·一般计算时间复杂度都是讲最差的情况,一般在计算时间复杂度的时候,忽略常数项和最低次项.因为当数据量很大的时候,这些项的计算时间其实差别不大.所以说真正的时间复杂度未必就是如此,只能说是一种粗算.但是和真实的时间复杂度的差别也微乎其微,所以时间复杂度真的只是标识一种状态.

·一篇写的很好的博客:https://blog.csdn.net/qq_41523096/article/details/82142747
如果是常量级的执行时间,比如O(2)实际上就是O(1)		算法不随数据量的增大而变化
如果是O(3n)实际上就是O(n)	去掉系数		算法随着数据量的增大而增大
在计算机中,如果算法是O(logn)	实际上就是以2为底,以n为真的对数函数.实际上如果是O(5logn)=O(logn)	算法的每次执行时间都比上一次要快
O(n^2)		算法每次的执行时间都比上一次要长,这里就是算法每次执行时间都是上次的两倍.

最常见:
O(1) < O(logn) < O(n) < O(n^2)

补充:
O(nlogn)、O(n^3)、O(m * n)、O(2^n)、O(n!)		n! n阶(高中学过的)

·经典例题1:
void fun(int n){			//计算该算法的时间复杂度
    int i = 1;
    while(i <= n){
	i = i * 2;
    }
}

·答案:实际上基本上一眼就能看出来是O(logn)
·解题方法:像这种题目一般就是看循环了多少次
①:循环条件 i<=n
②:i每次的变化 i = i * 2; 
③:假设每次执行了T次 当执行到第T次是,计算i的值是多少.在这里 i的初始值是1,每次执行i*2,那么当执行到T次时i的值是 i = 1*2的T方
④:将第三步的结果带入到第一步的条件中去,即 1*2的T方 = n 根据对数函数可知:t=log2n(以2为底n的对数) ∴O(n) = (logn) 最后一步是想办法把T转为n.

·另外:
-如果i的初始值是任意别的数,当我们计算时 由于这个数是放在系数的 最终根据约去系数的规定,对最终的时间复杂度的计算是没有影响的.
-如果是i = i * 3呢?实际上对结果还是不影响的.实际上我们可以通过上面的给的定义得知,O(logn)算法的特性就是随着和数据量的增大时,算法到最后的执行时间只会越来越快.

·结论:
对于for(int i = a; i <= n的b次方 ;i = i * c)这样的循环,它们的时间复杂度就是O(logn)
对于for(int i = a; i <= b * n ;i = i + c)这样的循环,时间复杂度就是O(n)

·思路:
上述说过PriorityQueue队列是一个有序队列,我们可以自定这个队列的排序规则,现在该队列的排序规则是按照路径的长度(权)由短到长进行排列.

首先我们需要维护一个Map,K是顶点;V是路径长度.

规则是这样的,首先将起始点加入Map中,然后将该起始点的所有邻接按权加入有序队列中.弹出队列中的第一个顶点,如果该顶点已经存在于Map中则重复弹出直到该顶点不在Map中.然后将该顶点加入Map中,然后将该顶点的所有邻接加入有序队列中(注意这里的邻接也必须是Map中没有的).

·步骤:
将A加入Map中;此时 Map{[A,0]}
将满足条件的A的邻接B和C加入有序队列中;此时Queue{[C,1],[B,5]}

将C,1加入Map;此时Map{[A,0],[C,1]}
将满足条件的C的邻接BDE加入有序队列中;此时Queue{[B,3],[B,5],[D,5],[E,9]}			//注意后面的权是和起始点A之间的距离

将B,3加入Map;此时Map{[A,0],[C,1],[B,3]}
将满足条件的B的邻接D加入有序队列;此时Queue{[D,4],[B,5],[D,5],[E,9]}

将D,4加入Map;此时Map{[A,0],[C,1],[B,3];[D,4]}
将满足条件的D的邻接EF加入有序队列中,此时Queue{[B,5],[D,5],[E,7],[E,9],[F,11]}

由于B已经存在于Map中,所以不能添加.D同理.此时将E,7加入Map;此时Map{[A,0],[C,1],[B,3],[D,4],[E,7]}
此时由于E的所有邻接皆不满足条件;此时Queue{[E,9],[F,11]}

由于E已经存在于Map中,所以不能添加.此时将F,11加入Map;此时Map{[A,0],[C,1],[B,3],[D,4],[E,7],[F,11]}
此时队列为空循环结束

此时从Map中根据顶点获得到的Value就是起始点距离各点之间的最短距离.

17.2回溯法(backtracking)

·什么是回溯算法?
回溯算法实际上一个类似枚举的搜索尝试过程,主要是在搜索尝试过程中寻找问题的解,当发现已不满足求解条件时,就"回溯"返回,尝试别的路径.回溯法是一种选优搜索法,按选优条件向前搜索,以达到目标.但当探索到某一步时,发现原先选择并不优或达不到目标,就退回一步重新选择,这种走不通就退回再走的技术为回溯法,而满足回溯条件的某个状态的点称为"回溯点".许多复杂的,规模较大的问题都可以使用回溯法,有"通用解题方法"的美称.

回溯算法的基本思想是:从一条路往前走,能进则进,不能进则退回来,换一条路再试.所以回溯算法和深度优先算法很想通.
回溯和递归是相辅相成的,一般所谓的回溯法就是递归函数.回溯法实际上是一种纯暴力的算法,但是用这种算法可以解决很多传统算法不能解决的问题.
应用场景:
1. 组合问题:假设有一个集合{1,2,3,4}找出这个集合大小为2的组合,那么它的结果有 <1,2> <1,3> <1,4> <2,3> <2,4> <3,4> 找出这些组合就是一个组合问题.
2. 切割问题:给定一个字符串问它有几种切割方式或者给定一个字符串如何切割才能保证它的字串都是回文字串问有几种切割方式.
3. 子集问题:假设有一个集合{1,2,3,4}找出这个集合的所有子集,那么它的结果有 <1> <2> <3> <4> <1,2> <1,3> <1,4> <2,3> <2,4> <3,4> <1,2,3> <2,3,4>
4. 排列问题:排列组合问题,排列相对于组合强调元素的顺序.假设有一个集合{1,2} 那么<1,2>就是它的组合 <1,2>和<2,1> 是它的排列.
5.棋盘问题:N皇后和解数独问题都是很复杂的棋盘问题

我们一般可以把回溯法抽象为一个树形结构(N叉树),树的宽度就是元素个数,树的深度就是每个节点递归的深度.
递归模板:
private void backtracking(args...){
	if(终止条件){
		收集结果;
		return 结果;
	}
	for(遍历集合中的每一个元素){
		处理节点;
		递归函数;(backtracking)

17.3贪心算法

·什么是贪心算法?
贪心算法（又称贪婪算法）是指,在对问题求解时,总是做出在当前看来是最好的选择.也就是说,不从整体最优上加以考虑,他所做出的是在某种意义上的局部最优解.

贪心算法不是对所有问题都能得到整体最优解,关键是贪心策略的选择,选择的贪心策略必须具备无后效性,即某个状态以前的过程不会影响以后的状态,只与当前状态有关.

·这个算法对于每个局部而言是最优的算法,从而整个整体也是最优的解法(这是因为整体最优解包含局部最优解)
贪心算法和动态规划的区别:
共同点：两者都具有最优子结构性质
不同点:
1) 贪心算法并不从整体最优上加以考虑,它所做的选择只是在某种意义上的局部最优解.
2) 动态规划将待求解的问题分解成若干个子问题,先求解子问题,然后从这些子问题的解得到原问题的解.并且动态规划中每步所做的选择往往依赖于相关子问题的解,因而只有解出相关子问题时才能做出选择(也就是动态规划并不关心局部最优解)
3) 动态规划算法通常以自底向上的方式解各子问题，而贪心算法则通常自顶向下的方式进行.

17.4动态规划(dp)

·什么是动态规划?
动态规划背后的基本思想非常简单.大致上,若要解一个给定问题,我们需要解其不同部分(即子问题),再根据子问题的解以得出原问题的解.动态规划往往用于优化递归问题,例如斐波那契数列,如果运用递归的方式来求解会重复计算很多相同的子问题,利用动态规划的思想可以减少计算量.
通常许多子问题非常相似,为此动态规划法试图仅仅解决每个子问题一次,具有天然剪枝的功能,从而减少计算量:一旦某个给定子问题的解已经算出,则将其记忆化存储,以便下次需要同一个子问题解之时直接查表.这种做法在重复子问题的数目关于输入的规模呈指数增长时特别有用.(这里就可以参考斐波那契数列)

19.KMP算法(字符串匹配算法)

·BF算法:
BF算法又称暴风算法,BF算法的思想就是将目标串S的第一个字符与模式串T的第一个字符进行匹配,若相等,则继续比较S的第二个字符和 T的第二个字符;若不相等,则比较S的第二个字符和T的第一个字符,依次比较下去,直到得出最后的匹配结果,时间复杂度为O(M*N)

·假如目标串是aaaaaaaab 模式串是aaaab 在这种极端的情况下,KMP算法可以帮助我们省略很多次的比较,因为我们每次比较的时候都是从失败的位置开始比较的.KMP的时间复杂度是O(m+n)

public String longestPrefix(String s) {
    int[] next = calNext(s + " ");
    return s.substring(0, next[s.length()]);
}

public int[] calNext(String pattern) {
    int[] next = new int[pattern.length()];
    next[0] = -1;
    int k = -1; // 前缀索引
    int j = 0; // 后缀索引
    while (j < pattern.length() - 1) {
        if (k == -1 || pattern.charAt(k) == pattern.charAt(j)) {
            k++;
            j++;
            next[j] = k;
        } else {
            k = next[k];
        }
    }
    return next;
}
