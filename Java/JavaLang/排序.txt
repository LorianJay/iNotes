·Comparable和Comparator的区别:
Comparable称之为内部比较器
Comparator称之为外部比较器
Comparable的耦合性比Comparator更高

使用Comparable的方法:
需要实体类实现Comparable方法,就像Integer这个类一样,排序的方法是直接写在当前这个类里面的
在自定义业务排序的实体类里面,就必须包含排序业务

使用Comparator的方法:
实体类与排序类解耦,实现在外部
如果在排序业务里面,一开始给的就是字节码文件没有源码,要排序的时候Comparable接口就无法使用,只能用Comparator接口
由于Comparator接口只有一个方法,所以我们可以不提供Comparator接口的实现类,直接使用Lambda表达式.javaSE搜索Lambda

容易忘记的返回值!!!:
这两个方法最后一定要有一个返回值,并且该返回值只有三种情况	>0	=0	<0
这三个返回值是用于排序的比较

·对于稳定性的解释:
这里使用选择排序和冒泡排序来进行比较:
如果有这样一个数组:int values = {1,9,6,7,3,5,9}

·选择排序:
·介绍:最简单但是最没用的排序算法,也有优化空间.为什么没用呢?时间复杂度很高,不稳定.基本上在工程上不会使用.
·为什么说时间复杂度很高呢?因为这种排序方法就算这个数组一开始就是有序的,他也要进行排序一遍.所以无需和有序没有差别.所以很没用.
·排序规则:基本思想试一遍又一遍过滤这个数组,找到这个数组最小的那个数,把它放到前面来.此时第一个数就是最小的那个数.当第二次在查找的时候,由于第一个数已经确认了,所以从第二个数开始查找.然后重复刚才的过程.
·具体实现:我们对每一个元素都提供一个下标,下标默认为0也就是第一个元素.让第一个元素和后面的元素进行比较.如果发现有一个元素的数值比当前的下标的数值小,那么就将下标的位置变为数值较小的那个元素的索引.然后接着进行比较.只要有一个数值较小那么下标的位置就变为当前较小数值的下标.当一次循环结束时,将当前下标下的元素和当前检索区域内的第一个元素的位置进行替换.
·具体实现代码:无解释版本

                }
            }
            int temp = values[j-1];
            values[j-1] = values[index];
            values[index] = temp;

·希尔排序:改进的插入排序
·设计思想:对一个数组提供个一个gap(间隔),把这个数组按间隔数拆分下来,然后第一次把每个查分下来的第一个数进行插入排序.然后再把第二个数进行插入排序.
这里用个具体的实例:
int[] values = {9,6,11,3,5,12,8,7,10,15,14,4,1,13,2};
我们提供这个间隔数为4,那么第一次比较应该就拿9,5,10,1来进行比较.
比较之后的结果应该就是:1,6,11,3,5,12,8,7,9,15,14,4,10,13,2.
第二次比较就应该拿6,12,15,13.以此类推.
当你排完这个数组之后,数组的情况应该是:1,6,2,3,5,12,8,4,9,13,11,7,10,15,14.
你会发现这个数组里大致的小的数都排到前面来了,大致的大的数都排到后面来了.
接下来,你需要缩小这个间隔数(gap).
但到了最后一定要以间隔为1排一遍.此时这最后一遍就相当于普通插入排序.

·那么这种算法为什么会比普通插入排序效率高呢?:
在上述提供的原始数组里面,如果这个一要排到最前面就要和之前的每一个数进行交换,但是如果用希尔排序,只需要挪很少的位置就可以排到最前面.因为他在间隔比较大的时候挪的次数比较少,他在间隔比较小的时候,挪的距离比较小.间隔比较大的时候交换的次数比较少,间隔比较小的时候移动规定距离比较短.所以希尔排序比普通的插入排序效率高.

·关于稳定性:由于希尔排序是跳着排所以他不稳定.

·关于gap数值的算法:
采用Kunth序列:
△h=1			解释:这个数列的最小值肯定是1
△h=3*h+1		解释:增加的时候就按照这个算法来增加,所以j就是1,4,13......我们所采用的那个序列数这个序列能对这个数组使			用的最大的序列.就是这个h必须要小于这个数组长度的3分之一.再大的就不适合使用了.因为如果再大3*h+1就大于整				个数组的长度了.
·具体代码:最终优化代码,采用Kunth序列
        int[] values = {9, 6, 11, 3, 5, 12, 8, 7, 10, 15, 14, 4, 1, 13, 2};
        int gap = 1;
        while (gap < values.length / 3) {
            gap = gap * 3 + 1;
        }
        for (; gap > 0; gap = (gap - 1) / 3) {
            for (int i = gap; i < values.length; i++) {
                boolean change = false;
                for (int j = i; j > gap - 1; j -= gap) {
                    if (values[j] < values[j - gap]) {
                        int temp = values[j - gap];
                        values[j - gap] = values[j];
                        values[j] = temp;
                        change = true;
                    }
                    if (!change) {
                        break;
                    }
                }
            }
        }
·关于效率:效率比插入排序更快.但是由于不稳定所以不怎么使用,好处就是最坏的情况和最好的情况时间复杂度差不多.不同的序列他们的时间复杂度是不同的.

·快速排序:经典的是单轴快排,优化的是双轴快排.要用到递归.
·单轴快排的设计思想:对于一组数据,首先找到一个数据作为轴(标准),这个轴可以是随机的.找到这个轴之后的设计思想是,比这个轴小的都排到前面去,比这个轴大的都排到后面去.当然这时排在轴前面的虽然都是比这个轴小的,但是这些个小的并没有按照顺序排好.同样比轴大的也没有排好顺序.此时你可以理解这个数组被分成了三部分,第一部分是比这个轴小的,第二部分是和这个轴一样大的,第三部分是比这个轴大的.然后再对较小部分和较大部分进行同样的运算规则.一直算下去,直到只剩下一个的时候.
·具体实现:
对于一个数组,从数组的左边(从零开始)和数组的右边(从最后开始)同时找数,从左往右找的时候,一直找到第一个比轴大的数,从右往左找的时候一直找到比轴小的数,这两个数一旦找到这两个数直接做交换.
·单轴快排的实现代码:最终版本,使用迭代器,效率非常高,和归并排序有的一拼.
public class Quick {
    public static void main(String[] args) {
        int[] values = {};
        sort(values, 0, values.length - 1);
    }

    public static void sort(int[] values, int leftBound, int rightBound) {
        if (leftBound >= rightBound)
            return;
        int mid = partition(values, leftBound, rightBound);
        sort(values, leftBound, mid - 1);
        sort(values, mid + 1, rightBound);
    }

    public static int partition(int[] arr, int leftBound, int rightBound) {
        int pivot = arr[rightBound];
        int left = leftBound;
        int right = rightBound - 1;
        while (left <= right) {
            while (left <= right && arr[left] <= pivot)
                left++;
            while (left <= right && arr[right] > pivot)
                right--;
            if (left < right) {
                int temp = arr[left];
                arr[left] = arr[right];
                arr[right] = temp;
            }
        }
        int temp = arr[left];
        arr[left] = arr[rightBound];
        arr[rightBound] = temp;
        return left;
    }
}

·双轴快排:Arrays.sort()jdk就是采用的双轴快排.
·设计思想:与单轴快排不同的是,双轴快排我们需要找两个数,把整个数组分成三个区域.比第一个轴小的数放左边,大于等于第一个轴,小于等于第二个轴放中间,大于第二个轴放后面.保障的前提是第一个数要比第二个数小.
·具体实现:
记录几个区域.小于第一个数的区域为小于区,称之为less.还有一个区域也是在这个位置上,大于等于第一个轴,小于等于第二个轴.称之为mid区.
比第二个轴还要大的位置称之为more区.然后数组没读一个数出来看这个数处于哪个位置.比如它在less位置,那你less的值就要+1.同样你的mid值也要衡量.如果是more区域那么你的值也要加.整体的算法过程就是这三个区域不断的吃掉中间还没有被处理的数的过程.
·具体实现的代码就不敲了,这个算法非常的复杂,是java集合的创始人写出来的算法,大牛的算法,直接用就行了.
·对于java集合的Arrays.sort()双轴快排的源码分析:
如果你这个数组的长度小于286个数,那么就去使用Timsort,否则使用非归并排序
在Timsort方法里(使用Timsort的过程中),他把这个数组分成好多小块,如果这个块数小于67块,那么才可以使用Timsort(改进的归并排序).
如果不适合用Timsort,就用非归并排序.
在非归并排序里,如果这个数组块数小于47,那么就直接使用插入排序的优化版本(双插入排序).否则就用快速排序(此时还不能确定是单轴快排还是双轴快排).
此时正式进入快速排序(此时要根据找轴结果来进一步分析,这个数组是适用于单轴的还是双轴的)
如何找轴?首先算法数组长度的七分之一,找到数组的中点作为一个轴,从中点减去这个范围(数组长度的七分之一).找到一个轴.再减这个范围找到一个轴,.然后从中点加上这个范围找到一个轴,再加上这个范围找到一个轴.这样就找到了五个数.这样就找到了E1,E2,E3,E4,E5.
把这五个轴进行排序,E1<E2<E3<E4<E5
然后对这五个轴进行判断,看是否相等,如果这五个轴都不相等时
使用第二个和第四个这个数作为轴.即E2和E4
此时有了两个轴,就可以执行上述具体实现里面的后面的操作.进行递归等待.
如果这五个数里面有相等的情况时,使用单轴快排
整体的这个数组的可能相等的数可能比较多,所以这时候我们能用一个轴来排(单轴快排),但是他用的是三相划分法来做的整个快速排序
此时这三个区域,第一个区域是小于这个轴的,第二个区域是等于这个轴的,第三个区域是大于这个轴的.(这个算法叫做Duch National Flag,荷兰国旗问题)

·计数排序:
·介绍:非比较排序,桶排序思想的一种.适用于数组量大,但是范围小.比如某大型企业数万名员工年龄排序.如何快速得知高考排名(腾讯面试).
解释一下高考,因为高考总分是固定的(范围很小),但是每年的考生有很多(数量大).
·设计思想:我们创建一个新的数组用来计数,数组的长度是可以取值的范围的长度,比如数值的范围是[0,60],那么数组的长度就是61.
然后我们去读取这个数组,比如这个数组是0,2,3,1,0,1.每当我们读到一个0的时候,就把数组索引为0的那个元素数值+1.如果我们读到1,就把索引为1的元素的数值+1,读到什么数,就把对应的数组的数值+1.相当于我们统计了每个数出现了多少次,然后再把这个数作为索引,把出现的次数存到新的数组对应的索引上.这个新的数组就叫计数数组.然后我们只要把计数数组里面的值依次取出来就可以了.
·问题:如果这个数组的取值范围不是从零开始,该怎么办?而且现在这种算法是不稳定的,怎样变成稳定的呢?
·关于改进:在第一次遍历新数组之后,我们把这个新的数组再遍历一次,从索引1开始遍历每次遍历把当前索引对应的值加上前一个元素对应的值.此时这个计数数组就记录着对应的每一个下标值,他的最后哪一位应该在什么位置上.(当然这个索引还需要-1).
·具体代码:最终优化代码
    public static int[] sort(int[] arr, int max) {
        int[] result = new int[arr.length];
        int[] count = new int[max];
        for (int i = 0; i < arr.length; i++) {
            count[arr[i]]++;
        }
        for (int i = 1; i < count.length; i++) {
            count[i] = count[i] + count[i - 1];
        }
        for (int i = arr.length - 1; i >= 0; i--) {
            result[--count[arr[i]]] = arr[i];
        }
        return result;
    }
·优化:现在的数组的取值范围任意,而且是稳定的.
·使用注意事项:如果你发现这个排序没有排序成功.注意当前的这个方法并不会排序传入的数组,而是有一个结果数组.所以你需要让原数组的内存等于返回值,注意第二个新参直接填入数组取值范围的最大值.
·特点:计数排序是非比较排序、适用于特定问题,也就是对源数据有要求.
·关于效率:在特定的情况下是非常快的,比目前已知的所有排序都快.比双轴快排还快.
}

·基数排序:
·介绍:非标排序,桶思想的一种,多关键字排序.
·设计思想:例如这里有一个数组:
int[] values = {421,240,115,532,305,430,124};
然后按照他们的个位数进行排序:240,430,421,532,124,115,305	其中{240,430}、{421}、{532}、{124}、{115,305}分别为一个桶
当个位数排好的情况下再来排十位数:305,115,421,124,430,532,240	其中{305}、{115}、{421,124}、{430,532}、{240}为一个桶
当十位数排好的情况下再来排百位数:115,124,240,305,421,430,532	其中{115,124}、{240}、{305}、{421,430}、{532}
·细节把控:这里的数组都是三位数,如果有一个四位数或者两位数该怎么办呢?反正排序都是从最大的数开始,比如从一个四位数1122开始,也是从各位开始,对于比他小的数只要在前面填上一个零即可.
·具体代码:使用低位优的最终版本
    public static int[] sort(int[] arr) {
        int[] result = new int[arr.length];
        int[] count = new int[10];

        int maxMath = 0;
        int index = 1;
        for (; index < arr.length; index++)
            maxMath = arr[maxMath] < arr[index] ? index : maxMath;
        index = 0;
        maxMath = arr[maxMath];
        for (; maxMath > 0;index++)
            maxMath /= 10;

        for (int i = 0; i < index; i++) {
            int division = (int) Math.pow(10, i);
            for (int j = 0; j < arr.length; j++) {
                int num = arr[j] / division % 10;
                count[num]++;
            }
            for (int m = 1; m < count.length; m++) {
                count[m] = count[m] + count[m - 1];
            }
            for (int n = arr.length - 1; n >= 0; n--) {
                int num = arr[n] / division % 10;
                result[--count[num]] = arr[n];
            }
            System.arraycopy(result, 0, arr, 0, arr.length);
            Arrays.fill(count, 0);

        }
        return result;
    }
·关于效率:效率也非常块,但是和计数排序比起来还是,慢一点.所以计数排序用的的基数排序多
·特点:
△可以排字符串
△本质上是一种多关键字排序
△有低位优先和高位优先两种
  △LAS MSD(Least Significant Digif first)
  △MSD属于分治的思想
△百度百科的程序有问题
 △空间复杂度变高

·桶排序:
·介绍:普通思想上的桶排序用的不是特别多,用的比较多的是桶排序的特殊例子,计数排序和基数排序.但其实计数排序用的最多.
·设计思想:
例如这里有一个数组:double values[] = {0.0,0.12,0.18,0.93,0.45,0.76,0.89,0.03,0.55,0.98,0.67,1.0};
现在假如把这12个小数分成四个桶,当然桶的个数可以你自已定
第一个桶存放数的区间为:[0.0,0.25)
第二个桶存放数的区间为:[0.25,0.5)
第三个桶存放数的区间为:[0.5,0.75)
第四个桶存放数的区间为:[0.75,1.0]
遍历数组找出最大值和最小值,计算最大值和最小值的差值,来得到每个桶的范围是多少.然后遍历数组里面的每一个元素,看这个元素属于哪个桶,然后把它放进这个桶里.放完之后再在每个桶里进行单独的排序.全部拍完之后再把每个桶之间合并起来,最后得到排完序的结果.
·需要注重的点:
每个桶的大小该是多少?
如果使用的是数组,并且你的数据排布的不是那么分散,基本都在0.0-0.25之间,那么你的桶的大小基本上要和原数组大小差不多,当然你也可以使用ArrayList,用它的自动扩容,但是这样会带来性能上的损失.
如果使用链表,那对于大小就不需要分配那么多了.但是如果使用链表,则对于链表的排序将非常麻烦,你需要使用冒泡排序的思想一次次的去遍历这个链表,那么效率将非常低.
·常见版本:
一般是将ArrayList作为桶,先把数据装进去之后,在使用归并排序或者快速排序进行排序.
}
