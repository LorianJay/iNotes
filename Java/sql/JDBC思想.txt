·介绍:							JDBC
JDBC(Java DataBase Connectivity,java数据库连接)是一种用于执行SQL语句的Java API，可以为多种关系数据库提供统一访问，它由一组用Java语言编写的类和接口组成。JDBC提供了一种基准，据此可以构建更高级的工具和接口，使数据库开发人员能够编写数据库应用程序，同时，JDBC也是个商标名。
sun公司由于不知道各个主流商用数据库的程序代码,因此无法自已写代码连接各个数据库,因此sun公司决定,自已提供一套api,范式数据库想与java进行连接的,数据库厂商自已必须实现jdbc这套接口.而数据厂商的jdbc实现,我们就叫他此数据库的数据库驱动.
·JDBC的本质:
(首先要加载驱动管理器)实际上就是通过java连接mysql服务器(scoket连接),然后通过java向mysql发送一些sql语句,mysql返回结果给java.

·Driver接口:
Driver接口由数据库厂家提供,对于java开发者而言,只需要使用Driver接口就可以了
在编程中要连接数据库,必须先装载特定厂商的数据库驱动程序.不同的数据库有不同的装载方法.
驱动:就是各个数据库厂商实现的sun公司提出的jdbc接口.即对Commection等接口的实现类的jar文件
-装载mysql驱动:
Class.forName("com.mysql.jdbc.Driver");
·需要引入的jar包:
compile group: 'mysql', name: 'mysql-connector-java', version: '8.0.16'

·使用到的数据表:

name varchar 不为null
password varchar 不为null
regTime datetime 不为null

2.Statement:
创建Statement类:通过刚才创建好的Connection类的createStatement()方法,返回一个Statement类

常用方法:

executeUpdate(String sql);	sql:sql语句(只限于select/delete/update)	返回结果的行数(返回值:int)
一般查询用executeQuery,非查询用executeUpdate.execute用的很少.

Statement statement = connection.createStatement();

特点分析:
1.如果这里的赵六这个人名不是事先定义好的,而是通过客户端那边传过来的,作为一个变量保存的话.就很不方便
  String name="赵六";

  String id= 5+" or 1=1;";	参数不规范,造成删库.
  String url = "delete from user where id="+id;

3.PreparedStatement:

常用方法:

executeQuery();			确认提交sql语句(只限于select),并且返回ResultSet接口
executeUpdate();		确认提交sql语句(只限于select/delete/update),返回更新(操作)的行数(返回值:int)

占位符:
此时我要传入的构造参数是:

5.ResultSet:
创建ResultSet的方法:通过Statement或其子类的executeQuery()方法.

类方法:
next()		将当前结果集的游标指向下一个.如果指向的内容不为空则范围true否则返回false(返回值类型:布尔值)
getInt(int index)	取出当前游标指向的行的第几列的int值	index:指代具体的列,要和表里面类的类型对应.返回int值
getString(int index)	取出当前游标指向的行的第几列的String值	index:指代具体的列,要和表里面类的类型对应.返回String值	
getObject(int index)	返回Object类
getClob(String columnLabel)	取出当前游标指向的行的第几列的String值	columnLabel:指代具体的列,就是以字符串指代

7.完整代码:
public class Demo4 {
    public static void main(String[] args) {
	//声明在外,便于关闭
        Connection connection = null;
        PreparedStatement preparedStatement = null;
        ResultSet resultSet = null;
        try {
            Class.forName("com.mysql.cj.jdbc.Driver");
            connection = DriverManager.getConnection("jdbc:mysql://localhost:3306/testjdbc?serverTimezone=UTC ", "root", "13218526138");

            String sql = "select * from user where id>?;";     //?:占位符
            preparedStatement = connection.prepareStatement(sql);
            preparedStatement.setInt(1, 2);  //将id大于2的记录查出
            resultSet = preparedStatement.executeQuery();
            while (resultSet.next()) {
                int id = resultSet.getInt(1);
                String name = resultSet.getString(2);
                String psssward = resultSet.getString(3);
                Date reGTime = resultSet.getDate(4);
                System.out.println(id + "---" + name + "---" + psssward + "---" + reGTime);
            }

        } catch (ClassNotFoundException e) {
            e.printStackTrace();
        } catch (SQLException e) {
            e.printStackTrace();
        } finally {
            if (resultSet != null) {		记得加上if判断,防止空指针异常
                try {
                    resultSet.close();
                } catch (SQLException e) {
                    e.printStackTrace();
                }
            }

            if (preparedStatement != null) {
                try {
                    preparedStatement.close();
                } catch (SQLException e) {
                    e.printStackTrace();
                }
            }

            if (connection != null) {
                try {
                    connection.close();
                } catch (SQLException e) {
                    e.printStackTrace();
                }
            }

        }
    }
}


8.批处理(Batch):
·介绍:对于大量的批处理,建议使用Statement,因为PreparedStatement的预加载空间有限,当数据量特别大时,会发生异常.
常用方法:
addBatch(String sql)	sql:sql语句 调用该方法不会立即执行,而是会添加一条sql语句,等待统一执行
executeBatch()		统一执行所有的sql语句 	返回一个int类型的数组

插入20w条数据示例:

//1.设置事物手动提交:
connection.setAutoCommit(false);

//2.创建Statement对象
statement = connection.createStatement();

//3.增加批量处理的语句:
for (int i = 0; i <20000 ; i++) {
	Random random = new Random();
 	statement.addBatch("insert into user(id,name,password,regTime) values("+i+",'People"+i+"','"+random.nextInt(999999)+"',now());");
}

//4.统一处理sql语句:
statement.executeBatch();

//5.提交事务:
connection.commit();

特点:
效率高

1.CLOB:
·介绍:
(1):用于存储大量的文本数据
(2):大字段有些特殊,不同数据库处理的方式不一样,大字段的操作常常是以流的方式来处理的.而非一般的字段,一次即可读出数据.
·前提:
这里我创建了一个新的表text,表信息:
id int 主键约束	自增约束
myInfo	text	
·目录:
1.通过IO流读取字符存储到mysql里:
2.读取mysql里的大文本保存为文件
·类方法:
getCharacterStream()			返回当前Clob所储存的大文本流	返回值类型:Reader

·PreparedStatement类的方法:
setClob(int parameterIndex, Reader reader)	parameterIndex:索引id	reader:IO流

·获取Clob对象的方法:
通过ResultSet类的getClob(String columnLabel)方法.结果返回一个Clob对象



1.通过IO流读取字符存储到mysql里:
//1.创建PreparedStatement类:
String sql = "insert into text(myInfo) values(?);";
preparedStatement = connection.prepareStatement(sql);

//2.通过IO读取文件
br = new BufferedReader(new FileReader("F:\\存放Java工程文件\\文件储存位置\\分割母文件utf8.txt"));
          
//3.将流读取到的内容设置到占位符的内容:
preparedStatement.setClob(1, br);

//4.确认提交:
preparedStatement.execute();

注意点:读取的文本的内容,必须与表中text列的编码集一致,都为UTF8.

2.读取mysql里的大文本保存为文件:

//1.确认执行并获取结果集:
String sql = "select myInfo from text where id=1;";
preparedStatement = connection.prepareStatement(sql);
resultSet = preparedStatement.executeQuery();

//2.循环结果集
while (resultSet.next()){
		Clob clob = resultSet.getClob("myInfo");        //通过列名获取到Clob对象
                reader = clob.getCharacterStream();            //得到Reader流,获取到reader之后,就可以正常显示这里面的内容了
                int temp = 0;
                while ((temp=reader.read())!=-1){
                    System.out.print((char) temp);
	}
}



2.BLOB:
·介绍:
(1):用于存储大量二进制数据
(2):大字段有些特殊,不同数据库处理方式不一样,大字段的操作常常是以流的方式来处理的.而非一般的字段,一次即可读出数据
·前提:
在text表里创建新的字段:
Image 类型:blob		用来存储一张图片
·mysql中相关的类型:
tinyblob:最大长度为255个字节blob列
blob:最大长度为65535字节的blob列
mediumblob:最大长度为16777215字节的blob列
longblob:最大长度为4294967295字节的blob列
·BLOB方法:
getBinaryStream()	返回当前Clob所储存的二进制数据	返回值类型:InputStream	

·PreparedStatement类关于BLOB的方法:	
setBlob(int parameterIndex, InputStream inputStream)		parameterIndex:索引	inputStream:字节流

·通过ResultSet类获取Blob对象:
getBlob(String columnLabel)		传入列名获得Blob对象


·目录:
1.通过BLOB存储任意文件
2.读取数据库中的BLOB文件

1.通过BLOB存储任意文件:

//1.创建PreparedStatement对象:
String sql = "insert into text(Image) values(?)";
preparedStatement = connection.prepareStatement(sql);

//2.通过IO流读取文件
bis = new BufferedInputStream(new FileInputStream("F:\\存放Java工程文件\\文件储存位置\\copy.jpg"));

//3.通过setBlob()方法,修改占位符为流内容
preparedStatement.setBlob(1,bis);

//4.确认执行
preparedStatement.execute();


2.读取数据库中的BLOB文件

//1.创建PreparedStatement对象:

while (resultSet.next()){
                Blob blob = resultSet.getBlob("Image");
                InputStream binaryStream = blob.getBinaryStream();
                int len = 0;
                while ((len=binaryStream.read())!=-1){
                    System.out.print((char)len);		这里只是随便输出的内容,实际情况中我们还可以通过输出流输出出来
	}
}
