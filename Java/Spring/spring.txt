							Spring
·介绍:
1.什么是spring:
spring是一个开源的框架,是一个轻量级的java开发框架.(轻量级:与EJB比较,依赖资源少、消耗的资源少)框架主要优势之一就是其分层架构.spring的核心是控制反转(IOC)和面向切面(DI).简单来说,spring是一个分层的javaSE/EE轻量级开源框架.(一站式:每一层都提供一个解决方案)

/*
分层:
web
service:spring
dao:mybatis、hibernate、jdbcTemplate、spring-data
*/

2.spring核心:
控制反转(IOC)和面向切面(DI)

3.spring优点:
方便解耦,简化开发(高内聚低耦合)
 spring就是一个大工厂,可以将所有对象创建和依赖关系维护,交给spring管理
 spring工厂生产的所有东西都是bean
AOP编程的支持
 spring提供面向切面编程,可以方便的实现对程序进行权限拦截、运行监控等功能.
声明式事务的支持:
 只需要通过配置就可以完成对事务的管理,而无需手动编程
方便程序的测试
 spring对junit4支持,可以通过注解方便的测试spring程序
方便集成各种优秀的框架
 spring不排斥各种优秀的开源框架,其内部提供了堆各种优秀的框架的直接支持.(如:hibernate、mybatis等)
降低javaEEAPI的使用难度
spring对javaEE中一些非常难用的API(JDBC、javaMail、远程调用),都提供了封装降低这些API的使用难度.

/*
什么是高内聚低耦合:
高内聚:一样的内容存放到一起,比如dao层我只存放操作数据库的内容
低耦合:能不互相调用就不互相调用.如果一个层面调用了另一个层面就是耦合.
*/

2.DI
·介绍:
依赖注入(组合)
/*
is a :是一个,继承
has a:有一个,依赖(组合)
*/

·解释:
实际上很简单,之前基础不是说过 组合类似继承吗. 依赖注入就是组合.
组合(依赖):一个对象依赖另一个对象
注入:通过setter方法进行另一个对象实例设置.
但这些都只是没有用spring的方法,现在用spring不需要new(IOC),不需要set(DI).你set的操作通过spring处理.

提供BookService接口和实现类
提供BookDao接口和实现类
其中BookService依赖注入BookDao

BookDao接口:
public interface BookDao {
    void addBook();
}

BookDao实现类:
public class BookDaoI implements BookDao{
    @Override
    public void addBook() {
        System.out.println("add Book");
    }
}

BookService接口:
public interface BookService {
    void addBook();
}

BookSerivce实现类:
public class BookServiceI implements BookService{
    private BookDao bookDao;	$1

    public void setBookDao(BookDao bookDao) {
        this.bookDao = bookDao;
    }

    @Override
    public void addBook() {
        this.bookDao.addBook();
    }
}

另外这种声明,虽然BookService类引用了BookDao类.表面上看不同层之间相互调用,好像违反了高内聚低耦合.但声明的是BookDao接口,所以你BookDao中代码发生改变和BookService没有关系,是根据你所传入的具体对象来执行的.所以这里解耦了.

<property>:注入标签
name:组合的属性名称和 $1对应 你需要设置当前实例里面的哪个属性
ref:设置的实例对象	注意这个值是xml里面的id值.而不是类名

4.2:Bean种类
·介绍:
普通Bean:之前的操作都是普通Bean.<bean id="" class=""></bean> spring底层直接创建实例,并返回.

FactoryBean:是一个特殊的Bean,具有工厂生产对象能力,只能生产特定的对象.
Bean必须实现Factory接口,此接口提供方法getObject() 用于获得特定的bean.
<bean id="" class="FactoryBean"></bean> 先创建FactoryBean实例,调用getObject()方法,并返回方法的返回值.

·BeanFactory和FactoryBean对比:
BeanFactory:工厂,用于生成任意Bean.生成的解释普通Bean.
FactoryBean:被实例出来本身就是一个特殊Bean,用于生成另一个特殊的Bean.例如:ProxyFactoryBean(Proxy:代理) 首先这是个工厂bean,然后这个工厂bean用于生产代理.<bean id="" class="...ProxyFactoryBean">获得代理对象实例.AOP使用

4.3:作用域
·介绍:
作用于:用于确定spring创建bean的实例的个数.

类别 		说明

singleton	在springIOC容器中仅存在一个Bean实例,Bean以单例方式存在,默认值.
prototype	每次从容器中调用Bean时,都会返回一个新的实例,即每次调用getBean()时,相当于执行new xxxBean()而且多利没有生命周期
request		每次HTTP请求都会创建一个新的Bean,该作用域仅适用于WebApplicationContext环境
session		同一个HTTP Session共享一个Bean,不同Session使用不同Bean,仅适用于WebApplicationContext环境
globalSession	一般用于portlet应用环境,该作用域仅适用于WebApplicationContext环境

·取值:
singleton	单例,默认值
prototype	多例,没执行一次getBean将获得一个实例.例如:Struts整合spring,配置action多例

·配置信息
<bean id="" class="" scope="">		通过scope来指定当前bean的作用范围

4.4.2:BeanPostProcessor后处理Bean
·介绍:
spring提供一种机制,只需要实现此接口BeanPostProcessor,并将实现类提供给spring容器,spring容器将自动执行,在初始化方法前执行before(),在初始化方法后执行after().通过<bean class="">来配置这个类
spring提供工厂钩子,用于修改实例对象,可以生成代理对象,是AOP底层.
·模拟:
A a = new A():

a.init();
a = B.after(a);
a.userAdd();		//生成代理对象,目的在目标方法执行前后执行代码(例如:开启事务、关闭事务)
a.Destroy();

MyBeanPostProcessor:
public class MyBeanPostProcessor implements BeanPostProcessor {
    /**
     *
     * @param bean  	回调,传入的实例对象
     * @param beanName 该实例对象的名称
     * @return
     * @throws BeansException
     */
    @Override

        System.out.println("后方法"+beanName);
        return bean;
    }

    @Override
    public Object postProcessBeforeInitialization(Object bean, String beanName) throws BeansException {
        System.out.println("前方法"+beanName);
        return bean;
    }

}

现在使用代理,实现执行实例工厂代码的前后执行代码

在MyBeanPostProcessor类的postProcessAfterInitialization方法:
    @Override
    public Object postProcessAfterInitialization(Object bean, String beanName) throws BeansException {
        System.out.println("后方法" + beanName);
        /*
        参数一:类加载器 这里是:当前类的类加载器
        参数二:目标类实现的接口
        参数三:InvocationHandler接口的实现类.实现该接口的invoke方法.这里用了Lambda表达式
         */
        return Proxy.newProxyInstance(MyBeanPostProcessor.class.getClassLoader(),
                bean.getClass().getInterfaces(),
                /*
                参数一:
                参数二:调用当前方法的方法(Method对象)
                参数三:调用该方法的方法的参数,是一个数组
                 */
                (Object proxy, Method method, Object[] args) -> {
                    System.out.println("开启事务");		//业务代码
                    /*
                    invoke()
                    参数一:真实对象,就是bean
                    参数二:args
                     */
                    Object obj = method.invoke(bean,args);	//真正执行的代码,实际上就是通过反射执行 调用该方法的方法
                    System.out.println("关闭事务");		//业务代码
                    return obj;
                });
    }

6.1.1:AOP介绍
·在软件业，AOP为Aspect Oriented Programming的缩写，意为:面向切面编程，通过预编译方式和运行期动态代理实现程序功能的统一维护的一种技术。AOP是OOP(面向对象)的延续，是软件开发中的一个热点，也是Spring框架中的一个重要内容，是函数式编程的一种衍生范型。利用AOP可以对业务逻辑的各个部分进行隔离，从而使得业务逻辑各部分之间的耦合度降低，提高程序的可重用性，同时提高了开发的效率。

·AOP采用横向抽取机制,取代了传统的纵向继承体系重复性代码.

·经典应用:性能监视、事务管理、安全检查、缓存

·springAOP使用纯java实现,不需要专门的编译和类加载器,在运行期通过代理方式向目标类植入增强代码.

·AspectJ是一个基于Java语言的AOP框架,Spring2.0开始,springAOP引入对Aspect的支持,Aspect扩展了Java语言,提供了一个专门的编译器,在编译时提供横向代码的的植入.

·到底是用来干嘛的?
UserService类:				A类:(切面类)
addUser()	//各种方法		before(){//开启事务}
deleteUser();				after(){//提交事务}
updateUser();

现在我在每次执行UserService类的所有方法前后需要加上 开启事务和关闭事务的代码.此时就可以使用代理来完成.(和之前的Proxy类似)
实现这种不需要重写目标类的代理就可以称之为AOP编程

面在哪里?理解下面的AOP原理中Aspect.将PointCut对应的方法和advice对应的方法,两两相连.当这之间的联系越来越多的时候,就会逐渐由多条直线形成一个面.

6.1.2:AOP实现原理
·AOP底层采用代理机制进行实现
·接口+实现类:spring采用jdk的动态代理Proxy
·实现类:spring采用cglib字节码增强

6.1.3:AOP术语(重要)

1.target:目标类,需要被代理的类.例如:UserService
2.Joinpoint(连接点):所谓的连接点是指那些可能被拦截的点.点是指:例如:UserService的所有方法
3.PointCut(切入点):已经被增强的连接点(方法).例如:addUser()
4.advice(通知/增强):增强的代码.例如:after、before

public class MyBeanFactory {
    public static UserService createService() {
        //1.目标类
        UserService userService = new UserServiceI();
        //2.切面类
        MyAspect myAspect = new MyAspect();
        /*
        3.代理类:将目标类(切入点)和切面类(通知) 结合-->切面
        参数一:loader,类加载器动态代理类运行时创建,任何类都需要类加载器将其加载到内存
            一般情况下:当前类.class.getClassLoader();
        参数二:代理类需要实现的所有接口
            方式一:目标类的实例(通过引用变量).getClass().getInterfaces()   只能获得当前类实现的直接接口,无法获取间接接口.
            方式二:new class[]{UserService.class};
        参数三:InvocationHandler处理类.提供invoke方法,代理类每次一次执行方法时,代理类都会去调用invoke方法
            参数3.1:Object proxy:代理对象
            参数3.2:代理对象当前执行的方法(反射)
                执行方法名:method.getName()
                执行方法:method.invoke(对象,参数)
            参数3.3:Object[] args :方法的实际参数
         */
        UserService proxyService = (UserService) Proxy.newProxyInstance(MyBeanFactory.class.getClassLoader(),
                userService.getClass().getInterfaces(),
                (proxy, method, args) -> {

                    //执行前方法
                    myAspect.before();

                    //执行目标方法
                    Object obj = method.invoke(userService, args);

                    //后方法
                    myAspect.before();
                    return obj;
                });
        return proxyService;
    }
}

public class MyBeanFactory {
    public static UserServiceI createService() {
        //1.目标类
        UserServiceI userService = new UserServiceI();
        //2.切面类
        MyAspect myAspect = new MyAspect();
        //3.代理类

        //3.1核心类
        Enhancer enhancer = new Enhancer();
        //3.2确定父类
        enhancer.setSuperclass(userService.getClass());
        /*
        3.3设置回调函数 MethodInterceptor接口等效于InvocationHandler
        intercept方法等效于invoke方法  参数1、2、3与invoke是一样的.
        参数四:一般不用 强行翻译为:方法的代理
        */
        enhancer.setCallback(new MethodInterceptor() {
            @Override
            public Object intercept(Object proxy, Method method, Object[] args, MethodProxy methodProxy) throws Throwable {
                //前方法
                myAspect.before();
                //执行目标方法
                Object obj = method.invoke(userService, args);
                //执行代理类的父类方法,实际上就是执行目标类
                //methodProxy.invokeSuper(proxy,args);
                //后方法
                myAspect.after();

                return obj;
            }
        });
        //3.4创建代理
        UserServiceI proxyService = (UserServiceI) enhancer.create();
        return proxyService;
    }
}

6.6.3:AspectJ通知类型
·介绍:
aop联盟定义的通知类型,具有特定规定接口,必须实现从而确定方法的名称.
aspectJ通知类型,只定义了类型名称,已经方法格式.
个数:一共6中,知道5种,掌握1种.

before:前置通知(应用:各种校验)
在方法执行前执行,如果通知抛出异常,阻止方法运行.

afterReturning:后置通知(应用:常规数据处理)
方法正常返回后执行,如果方法中抛出异常,通知无法执行.
必须在方法执行后才执行,所以可以获得方法的的返回值.	
也就是说目标方法执行后,才会执行afterReturning指定的方法,如果目标方法有返回值,这个返回值会作为参数传递到后置方法里.

around:环绕通知(应用:十分强大,可以做任何事情)
方法执行前后分别执行,可以阻止方法的执行.
必须手动执行目标方法

afterThrowing:抛出异常通知(应用:包装异常信息)
方法抛出异常后执行,如果方法没有抛出异常,无法执行.

after:最终通知(应用:清理现场)
方法执行完毕后执行,无论方法中是否出现异常.

使用try catch块对环绕进行描述:	以下所有的事情,环绕都能做
try{
   //前置方法before
   //手动执行目标方法	
   //后置方法after
}catch(){
   //抛出异常 afterThrowing
}finally{
   //最终 after	
}

ProceedingJoinPoint接口方法:
getTarget() return Object;					//得到目标对象的Object类型的对象.
getSignature() return Signature;				//得到目标方法的方法签名
getArgs() return Object[];					//得到目标方法的所有参数.这些参数是有值,就是已经可以得到从前端传过来的值了.
proceed() return Object;					//执行目标方法!这个很重要!!!!返回目标方法的执行完的返回值.

getTarget:对于getTarget方法的详解,该方法返回Object对象.通过调用该方法的getClass方法得到目标对象的Class对象,有了Class就很灵活了.
getSignature:对于getSignature方法的详解,该方法返回Signature对象,可以通过该对象的getName方法得到目标方法的方法名称.

MethodSignature:该接口是Signature的子接口,可以直接将得到的Signature对象强转为MethodSignature
getName() return String;			//得到目标方法的方法名称
toString() return String;			//得到的结果是 [目标方法返回值的全限定名]+[目标方法的全限定名]+([目标方法的参数的全限定						名])

8.2:三个顶级接口:
·PlatformTransactionManager		平台事务管理器,spring要管理事务,必须使用事务管理器
进行事务配置时,必须配置事务管理器.
·TransactionDefinition:事务详情(事务定义、事务属性)	spring用于确定事务具体详情,例如:隔离级别、是否只读、超时时间等
进行事务配置时,必须配置事务详情.spring将配置项封装到该对象实例.
·TransactionStatus:事务状态,spring用于记录当前事务运行状态.例如:是否有保存点,事务是否完成.
spring底层根据状态进行相应操作.

8.2.1:PlatformTransactionManager	 事务管理器
·常见的事务管理器:
DataSourceTransactionManager		JDBC开发时的事务管理器,采用JdbcTemplate
HibernateTransactionManager		Hibernate开发时事务管理器,整合hibernate.

·API详解:
TransactionStatus getTransaction(@Nullable TransactionDefinition definition)
事务管理器通过"事务详情",获得"事务状态",从而管理事务.
void commit(TransactionStatus status)
根据状态提交
void rollback(TransactionStatus status)
根据状态回滚

8.2.2:TransactionStatus		事务状态		//和上面的顺序不一样

·API详解:
isNewTransaction();	返回值:布尔	是否有新事务
hasSavepoint();		返回值:布尔	是否有保存点
setRollbackOnly();	返回值:无	设置回滚
isRollbackOnly();	返回值:布尔	是否回滚
flush();		返回值:无	刷新
isCompleted();		返回值:布尔	是否完成

8.2.3:TransactionDefinition	事务详情

·API详解:
getName();		返回值:String		配置事务详情名称.一般方法名称.例如:save、add*等
isReadOnly();		返回值:布尔		是否只读(增删改是读写,查询是读)
getTimeOut();		返回值:int		获得超时时间
getIsolationLevel();	返回值:int		隔离级别
getPropagationBehavior();返回值:int		传播行为	(重点)

剩下一些常量:事务的传播特性 这些常量全都来源于TransactionDefinition接口
·传播行为:在两个业务之间如何共享事务.
PROPAGATION_REQUIRED		[默认值]
	支持当前事务,A如果有事务,B将使用该事事务
	如果A没有事务,B将创建一个新事务
PROPAGATION_SUPPORTS
	支持当前事务,A如果有事务,B将使用该事务
	如果A没有事务,B将以非事务执行.
PROPAGATION_MANDATORY
	支持当前事务,A如果有事务,B将使用该事务
	如果A没有事务,B将抛出异常

PROPAGATION_REQUIRES_NEW
	如果A有事务,将A的事务挂起,B创建一个新的事务
	如果A没有事务,B创建一个新的事务
PROPAGATION_NOT_SUPPORTED
	如果A有事务,将A的事务挂起,B将以非事务执行
	如果A没有事务,B将以非事务执行
PROPAGATION_NEVER
	如果A有事务,B将抛出异常
	如果A没有事务,B将以非事务执行
PROPAGATION_NESTED
	A和B底层采用保存点机制(手动开启事务),形成嵌套事务.相当于A事务中嵌套了一个B事务,当B事务执行完之后才执行A事务.

掌握:PROPAGATION_REQUIRED、PROPAGATION_REQUIRES_NEW、PROPAGATION_NESTED

public class AccountServiceI implements AccountService {

    //通过spring注入模板
    private TransactionTemplate transactionTemplate;

        this.transactionTemplate = transactionTemplate;

            @Override
            protected void doInTransactionWithoutResult(TransactionStatus status) {
                accountDao.out(outer, money);
                int i = 1 / 0;
                accountDao.in(inner, money);

    }

关键看@Transactional这个注解:
Propagation propagation() default Propagation.REQUIRED;			//指定事务的传播级别.Propagation是一个枚举
Isolation isolation() default Isolation.DEFAULT;			//指定事务的隔离级别.Isolation也是一个枚举
boolean readOnly() default false;					//指定事务是否为只读事务,不需要事务的方法就可以设置为true
int timeout();					//事务的超时时间,如果事务在该事件内还没有完成就回滚.
Class<? extends Throwable>[] rollbackFor() default {};		//当这个方法抛出什么异常的时候回滚事务.
Class<? extends Throwable>[] noRollbackFor() default {};	//当这个方法抛出哪些异常的时候不会滚事务.


Propagation:事务传播级别

首先什么是事务传播级别?事务传播级别是spring的特性,事务传播级别是针对方法而言的.
方法A是一个事务方法,方法B不是一个事务方法.那么现在方法A和方法B互相调用会是什么情况?
拿REQUIRED来体会体会,现在方法B是REQUIRED但是方法A不是事务方法.现在方法A内部调用了方法B,那么在调用方法B的时候方法B会创建一个新的事务.
假如现在方法A是事务方法,方法A调用了方法B.那么方法B就不会新建一个事务,而是和方法A的操作都共用一个事务.

REQUIRED			----如果A没有事务,B将创建一个新事务(默认)
SUPPORTS			假如方法A调用方法B的时候方法A没有事务,那么方法B就会以非事务执行(如果方法A有事务就共用一个事务)
MANDATORY			支持当前事务,A如果有事务,B将使用该事务----如果A没有事务,B将抛出异常
REQUIRES_NEW			如果A有事务,将A的事务挂起,B创建一个新的事务----如果A没有事务,B创建一个新的事务
NOT_SUPPORTED			如果A有事务,将A的事务挂起,B将以非事务执行----如果A没有事务,B将以非事务执行
NEVER				如果A有事务,B将抛出异常----如果A没有事务,B将以非事务执行
NESTED			A和B底层采用保存点机制(手动开启事务),形成嵌套事务.相当于A事务中嵌套了一个B事务,当B事务执行完之后才执行A事务.

Isolation:
READ_UNCOMMITTED		会脏读、不可以重复读、幻读
ISOLATION_READ_COMMITTED	不会脏读、会不可重复读、幻读
ISOLATION_REPEATABLE_READ	不会脏读、不可重复读 会幻读
ISOLATION_SERIALIZABLE		不会脏读、不可重复读、幻读
DEFAULT				使用当前数据库默认的隔离事务(mysql默认为READ_UNCOMMITTED)

//这里比较巧妙的一点是这里dao层要实现的接口正好和映射接口一致.当前接口是被配置在mapper.xml映射文件中的
public interface StudentMapper {

}

//7.1.4 Dao层接口的实现类:		//该类就是之前说的要继承SqlSessionDaoSupport
//这样搞的好处是为每一个 sql操作的java方法都提供一个实现方法.这是之前mybatis没有讲过的.之前写一个接口方法就用一个类的一个方法测试一下,其实完全可以用一个类全部实现这些查询的功能.到底层而言虽然是直接掉接口的方法,然后有返回值,但是调这个方法还是需要代码的.

public class StudentMapperImpl extends SqlSessionDaoSupport implements StudentMapper {
    @Override
    public Student queryStudentById(Integer i) {
        SqlSession sqlSession = super.getSqlSession();		//通过继承直接调用父类的方法.获取到SqlSession.
        StudentMapper mapper = sqlSession.getMapper(StudentMapper.class);
        Student student = mapper.queryStudentById(i);
        return student;
    }
}
